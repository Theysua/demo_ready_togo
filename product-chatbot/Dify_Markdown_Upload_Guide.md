# Dify 完美 Markdown 上传指引

当你使用 `consolidate.py` 合并出终极的 `Goodyear_Hybrid_Knowledge_Base.md` 之后，将这个极度纯净的 Markdown 文件上传给 Dify 时，正确的分段参数将是决定知识召回成败的最后一步。

### 第一步：选择正确的分块模式
在 Dify 的“文本分段与清洗”界面：
1. **不要选择“自动”**。
2. 左下角请选择：**【层级模式 (Hierarchical Model) / 父子块设定】**方案。
*(如果你的 Dify 版本较旧没有层级模式，直接选“自定义”即可)*

---

### 第二步：参数完美设定

#### 1. 父块设定 (Parent Chunk)
父块的目的是为了给大模型提供“全局视野”，让模型不管搜到什么细枝末节，都能通过附带的父块拿到整页产品的上下文（轮胎名、静音科技介绍等）。

*   **分段标识符**：`\n\n---\n\n## Page` 
    *(说明：我们在 `consolidate.py` 脚本里，特意给每一页的开头都加上了这串超级分隔符。把这个设为标识符，Dify 就会无比精准地把手册的“每一页”切割成一个完整、绝对不会断裂的父块！)*
*   **分段最大长度**：`4000` 或 `8000`
    *(说明：既然我们按“页”切了，有些产品页内容很多，直接把 Token 长度开到最大，确保一整页的内容绝对不会因为超出 1000/2000 的紧凑限制而被强制劈开)*

#### 2. 子块设定 (Child Chunk)
子块的目的是“精准命中”。它是 Dify 向量搜索实际拿来比对的靶子。子块切得越准，用户搜 `105Y` 或者 `225/45R17` 时就越能瞬间召回。

*   **分段标识符**：`\n`
    *(说明：非常关键！因为我们的脚本把所有复杂的尺寸表格都变成了严格的 `| 315/30ZR21 | 105Y |` 单行格式。设置为 `\n` 是为了保证“切口绝对不会落在表格行中间”，绝不发生 `105Y` 被遗落到下一段的惨案)*
*   **分段最大长度**：`500` 或 `800`
    *(说明：子块不需要太大，只要能装下几行产品描述或者几行表格参数，能产生高浓度的检索相似度就行)*

---

### 第三步：测试与验收
全部设定好 -> 点击【保存上传】。

现在你可以试着在客服聊天框里问：
1. `"固特异 EAGLE F1 SUPERSPORT 到底包不包含 315/30ZR21 这个规格？它的载重指数是多少？"`
2. `"2023 亚太目录里，Wrangler 相关的轮胎历史里提到哪一年去了月球？"`

你会发现它现在的回复，将是你玩 RAG 以来最丝滑、最详尽、毫无遗漏的一次。因为机器终于读懂了机器该读的格式。
